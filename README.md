# نمودار حشر — راه‌انداز کامل (مرحله به مرحله)

این پروژه یه سایت زنده‌ست که سطح «حشر» هر عضو رو بین همه‌ی گوشی‌ها sync می‌کنه.
برای اینکه آنلاین بشه و یه لینک واقعی بگیری، باید این مراحل رو دنبال کنی.
هیچ‌کدوم نیاز به کارت بانکی ندارن.

---

## مرحله ۱ — ساخت دیتابیس در Firebase (محل ذخیره‌ی سطح‌های حشر)

1. برو به **console.firebase.google.com** و با اکانت گوگلت وارد شو.
2. روی **Add project** بزن.
3. یه اسم بده (مثلاً `hashr-grid`) → Continue → می‌تونی Google Analytics رو **خاموش** کنی (لازم نیست) → **Create project**.
4. وقتی پروژه ساخته شد، از منوی سمت راست (Build) برو روی **Realtime Database**.
5. روی **Create Database** بزن.
   - لوکیشن رو هر چی پیشنهاد داد بزن Next
   - وقتی پرسید با چه قوانینی شروع کنی، گزینه‌ی **Start in test mode** رو انتخاب کن (بعداً قانون امن‌تر رو خودم گذاشتم تو فایل `database.rules.json` که می‌تونی جاش بگذاری)
6. بعد از ساخته شدن، یه آدرس می‌بینی شبیه:
   `https://hashr-grid-default-rtdb.firebaseio.com`
   این رو جایی کنار بگذار — `databaseURL` همینه.

### گرفتن کلیدهای اتصال (Config)
1. روی آیکون چرخ‌دنده بالای صفحه (کنار "Project Overview") بزن → **Project settings**.
2. پایین صفحه‌ی General، بخش **Your apps** رو پیدا کن → روی آیکون **</>** (Web) بزن.
3. یه نام بده (مثلاً `hashr-web`) → **Register app**.
4. یه بلوک کد می‌بینی شبیه این:

```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "hashr-grid.firebaseapp.com",
  databaseURL: "https://hashr-grid-default-rtdb.firebaseio.com",
  projectId: "hashr-grid",
  storageBucket: "hashr-grid.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123",
};
```

5. این مقادیر رو دقیقاً کپی کن.

### وارد کردن کلیدها در پروژه
فایل `lib/firebase.js` رو باز کن و مقادیر `PASTE_YOUR_..._HERE` رو با مقادیر واقعی خودت جایگزین کن. نتیجه باید دقیقاً شبیه بلوک بالا باشه (با مقادیر خودت).

> ⚠️ این کلیدها برای دیتابیس عمومی پروژه‌ست (نه پسورد شخصی)، نشتشون خطر امنیتی جدی ندارد چون قوانین `database.rules.json` دسترسی رو محدود به سطح ۱ تا ۱۰ کرده.

## مرحله ۲ — بالا بردن کد روی GitHub

1. برو به **github.com** و وارد اکانتت شو.
2. روی دکمه‌ی سبز **New** (یا علامت `+` بالای صفحه → New repository) بزن.
3. اسمش رو بگذار `hashr-grid` → Public → **Create repository**.
4. توی صفحه‌ای که باز می‌شه، گزینه‌ی **uploading an existing file** رو بزن.
5. تمام فایل‌ها و پوشه‌های این پروژه رو (همه‌چی داخل پوشه‌ی `hashr-app`) بکش و توی همون باکس رها کن (drag & drop).
   - مهم: باید ساختار پوشه‌ها حفظ بشه (`pages/`, `lib/`, `styles/` و بقیه). اگه گیت‌هاب پوشه‌ها رو قبول نکرد، می‌تونی همه رو با هم زیپ کنی و آپلود کنی، یا از روش زیر (GitHub Desktop) استفاده کنی که ساده‌تره:
   - **روش ساده‌تر:** نرم‌افزار **GitHub Desktop** (desktop.github.com) رو نصب کن، با اکانتت وارد شو، گزینه‌ی "Add local repository" رو بزن و پوشه‌ی `hashr-app` رو انتخاب کن، بعد دکمه‌ی **Publish repository** رو بزن. این مطمئن‌ترین راهه.
6. بعد از آپلود، روی **Commit changes** بزن.

## مرحله ۳ — دیپلوی روی Vercel (گرفتن لینک نهایی)

1. برو به **vercel.com** → **Continue with GitHub**.
2. روی **Add New... → Project** بزن.
3. ریپازیتوری `hashr-grid` رو از لیست پیدا کن و **Import** بزن.
4. تنظیمات رو دست نزن (Vercel خودش تشخیص می‌ده Next.js هست)، فقط روی **Deploy** بزن.
5. حدود ۳۰–۶۰ ثانیه صبر کن. وقتی تموم شد، یه لینک می‌گیری شبیه:
   `https://hashr-grid.vercel.app`

این لینک رو می‌تونی مستقیم توی تلگرام برای دوستات بفرستی. هر کس بازش کنه، اسم خودش رو از صفحه‌ی ورود انتخاب می‌کنه و فقط سطح حشر خودش رو می‌تونه عوض کنه — و همه‌چی برای همه‌ی اعضا realtime sync می‌شه.

---

## اگه به مشکل خوردی

- **صفحه سفیده یا ارور Firebase می‌ده:** احتمالاً کلیدهای `lib/firebase.js` رو درست کپی نکردی، یا Realtime Database رو نساختی (مرحله ۱، قدم ۴-۵).
- **توی Vercel build fail می‌شه:** پیام ارور رو بخون، معمولاً یعنی یه فایل درست آپلود نشده یا یه فایل اضافه‌ست. می‌تونی پیامش رو برام بفرستی تا کمک کنم.
- **می‌خوای دومرتبه دیپلوی کنی بعد از تغییر کد:** کافیه فایل تغییریافته رو دوباره توی همون ریپازیتوری گیت‌هاب آپلود کنی (یا با GitHub Desktop کامیت و پوش کنی) — Vercel خودش به‌صورت خودکار دوباره دیپلویش می‌کند.
